<Tab value="Save tokens on my Claude Code / Cursor / Codex sessions">

Best for people running Claude Code, Cursor, or Codex directly and wanting the fastest path to lower token usage.

## 1. Install the proxy bundle

```bash
pip install "headroom-ai[proxy]"
```

## 2. Wrap your agent

```bash
headroom wrap claude

# Or another supported agent
headroom wrap cursor
headroom wrap codex
```

<Callout type="info" title="What wrap mode does">
Headroom starts the local proxy for you, routes your agent traffic through it, and keeps later compressions silent so the agent output stays clean.
</Callout>

## 3. Watch the first savings show up in the terminal

On the first saved-token request in the session, Headroom prints a one-time banner:

```text

▶ First compression: 17,765 -> 1,408 tokens  92% saved
  Transforms: SmartCrusher · CacheAligner
  ≈ $0.048 saved · session total so far: $0.048
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

When the wrapped session exits, Headroom prints the session total and points you at the next high-value step:

```text

 Run `headroom learn` to extract lessons from this session.
```

<Callout type="info" title="TypeScript SDK requires the proxy">
The TypeScript SDK sends messages to the local Headroom proxy for compression. Start it once before calling `compress()` from TypeScript:

```bash
pip install "headroom-ai[proxy]"
headroom proxy --port 8787
```
</Callout>

## 2. Call `compress()`

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```ts twoslash
import { compress } from 'headroom-ai';

const messages = [
  { role: 'system' as const, content: 'You summarize noisy support tickets.' },
  {
    role: 'user' as const,
    content: JSON.stringify({
      tickets: Array.from({ length: 300 }, (_, i) => ({
        id: i + 1,
        priority: i % 5 === 0 ? 'high' : 'normal',
        detail: `Verbose support ticket payload ${i + 1}`,
      })),
    }),
  },
];

const result = await compress(messages, {
  model: 'gpt-4o',
  baseUrl: 'http://localhost:8787',
});
```
</Tab>
<Tab value="Python">
```python
from headroom import compress
import json

messages = [
    {"role": "system", "content": "You summarize noisy support tickets."},
    {
        "role": "user",
        "content": json.dumps({
            "tickets": [
                {
                    "id": i + 1,
                    "priority": "high" if i % 5 == 0 else "normal",
                    "detail": f"Verbose support ticket payload {i + 1}",
                }
                for i in range(300)
            ]
        }),
    },
]

result = compress(messages, model="gpt-4o")
```
</Tab>

## What gets compressed

The biggest savings come from tool outputs, search results, logs, structured JSON, and other bulky context blocks. Headroom auto-detects the content type and routes it to the best compressor.

| Content type | Compressor | Typical savings |
|---|---|---|
| JSON arrays | SmartCrusher | 70-90% |
| Source code | CodeCompressor | 40-70% |
| Build/test logs | LogCompressor | 80-95% |
| Search results | SearchCompressor | 60-80% |
| Plain text | Kompress | 30-50% |
