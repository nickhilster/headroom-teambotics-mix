Design · Section 1: Quickstart page structure

The quickstart page gets restructured around a role selector. Everything below it adapts to the chosen track.

Current structure (problem)

quickstart.mdx — today
1. Install — pip / npm tabs
2. Compress messages — code example (library API, complex message array)
3. Send to your LLM — more code
4. Check your savings — console.log output (← aha moment, step 4)
— Alternative: proxy mode (buried at bottom)
→ Power user never sees wrap. App builder must parse proxy section. Both must read all 4 steps.

Proposed structure

quickstart.mdx — proposed
↑ Above the fold: role selector
  [ I use Claude Code / Cursor / Codex ]   [ I'm building an app ]

Track A — Agent User (3 steps, ~60 seconds)
  1. pip install "headroom-ai[proxy]"
  2. headroom wrap claude  (auto-starts proxy, wires Claude)
  3. Inline savings appear in your terminal ← aha moment, step 3
  → Next: headroom learn · persistent install · cross-agent memory

Track B — App Builder (3 steps, ~90 seconds)
  1. pip install headroom-ai / npm install headroom-ai
  2. compress(messages, model="…") — single function call, minimal example
  3. result.tokens_saved printed immediately ← aha moment, step 3
  → Next: proxy mode · SDK wrappers · LangChain · Vercel AI SDK

Key changes from today

  • Role selector is above the fold — first thing seen, not a buried "Alternative" section
  • Aha moment moves from step 4 → step 3 in both tracks
  • Power user path never sees library code; app builder path never sees wrap
  • Each track ends with a focused "next steps" pointing deeper into that persona's journey
  • Proxy mode stays prominent — it's step 2 of Track A, not an afterthought

Does this structure look right? Reply in terminal — then we'll move to the CLI savings output design.
