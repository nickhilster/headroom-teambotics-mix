Design · Section 2: CLI savings output

The aha moment for the Agent User track lives here — what the terminal shows on first compression.

Current state (problem)

$ headroom wrap claude — today
Starting proxy on :8787…
Launching claude…
[session runs silently — savings only visible via `headroom stats` after the fact]

→ User has no idea if Headroom is doing anything. Value is invisible during the session.

Proposed: inline savings banner on first compression

$ headroom wrap claude — proposed
✓ Headroom proxy running on :8787
✓ Claude Code wired — compression active
Tip: run `headroom stats` anytime to see savings
[claude session starts normally…]

App Builder equivalent: compress() return value

Python — first compress() call
result = compress(messages, model="gpt-4o")
# result already has .tokens_before / .tokens_after / .tokens_saved
# Quickstart step 3 just prints these — no extra work needed
print(f"Saved {result.tokens_saved} tokens ({result.compression_ratio:.0%})")

→ No CLI change needed for the App Builder track — the data is already on the result object. The quickstart just needs to show it at step 3 rather than step 4.

Design rules

  • Banner fires once per session — on first compression only, not every call (avoids noise)
  • Session-end summary is always shown — gives the cumulative picture when the session ends
  • Silent after first banner — subsequent compressions don't print anything; agent output is not polluted
  • Cost estimate is opt-in friendly — shown by default, suppressible via config if users don't want it
  • headroom learn nudge at session end — surfaces the feature without requiring users to discover it separately
